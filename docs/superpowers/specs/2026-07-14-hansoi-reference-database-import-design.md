# 한소이 레퍼런스 DB 적재 설계

## 목표

`hansoi_references` 폴더의 PNG 12장을 원격 `ai_sns` 데이터베이스의
`한소이`(`public_id = soi_film`) 비주얼 프로필 레퍼런스로 등록한다.

## 범위

- 애플리케이션 코드와 현재 레퍼런스 5장 제한은 변경하지 않는다.
- 12개 원본 파일을 S3의
  `pod/reference/character/019f421d-0311-730a-877c-31e29c5d71ac/` 경로에
  서로 다른 키로 업로드한다.
- 각 객체를 `opod.media`에 이미지 미디어로 기록하고 `uploaded_at`을 설정한다.
- 한소이의 `character_visual_profiles` 레코드가 없으면 생성한다.
- `hansoi_01.png`부터 `hansoi_12.png`까지 파일 번호 순으로 `sort_order`를
  10, 20, ..., 120으로 설정해 모두 연결한다.

## 데이터 흐름

1. 12개 파일의 형식, 크기, 해상도, SHA-256을 수집한다.
2. 기존 S3/DB에 동일한 적재 키 또는 URL이 있는지 확인한다.
3. S3 업로드 12건이 모두 성공한 뒤 원격 DB 트랜잭션을 실행한다.
4. 트랜잭션에서 미디어 12건, 비주얼 프로필, 레퍼런스 12건을 생성한다.
5. 원격 DB의 연결 수, 순서, URL과 S3 객체 접근성을 검증한다.

## 실패 처리

- S3 업로드가 하나라도 실패하면 DB 쓰기를 시작하지 않는다.
- DB 쓰기는 단일 트랜잭션으로 실행해 일부 레코드만 연결되는 상태를 막는다.
- 재실행 시 동일 적재 키를 기준으로 기존 레코드를 확인해 중복 삽입을 피한다.
- 현재 서비스는 5장 초과 세트를 저장하지 못하므로, 제한 변경 전 관리자 화면에서
  이 레퍼런스 세트를 수정하지 않는다.

## 성공 기준

- 원격 한소이 비주얼 프로필에 레퍼런스가 정확히 12건 존재한다.
- 12건의 `sort_order`가 10부터 120까지 파일 번호 순으로 매핑된다.
- 각 미디어가 PNG 메타데이터와 `uploaded_at`을 가지며 공개 URL에서 조회된다.
- 로컬 애플리케이션 소스의 레퍼런스 제한은 변경되지 않는다.
